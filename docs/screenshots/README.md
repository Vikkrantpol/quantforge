# QuantForge Screenshots

Store product screenshots for the README and GitHub repo in this folder.

Suggested capture order and filenames:

1. `01-quant-lab-overview.png`
   Main Quant Lab form showing symbol, period, interval, strategy, and execution settings.

2. `02-live-backtest-terminal.png`
   In-progress backtest with the terminal log visible while a job is running.

3. `03-results-overview-dashboard.png`
   Results overview page with total return, CAGR, max drawdown, win rate, and top-level cards.

4. `04-results-charts-equity-drawdown.png`
   Charts tab showing the equity curve and drawdown view.

5. `05-results-trade-log.png`
   Trades tab with entry, exit, quantity, PnL, and timestamps visible.

6. `06-history-dashboard.png`
   History page with multiple saved backtests and reopen flow visible.

7. `07-data-fetch-workspace.png`
   Data fetch workflow for downloading OHLCV data or loading CSV inputs.

8. `08-broker-config-fyers.png`
   Broker page with FYERS setup fields and connection workflow.

9. `09-multi-market-symbols.png`
   A screenshot that clearly shows India, US, and crypto symbols in one product flow.

10. `10-mobile-or-narrow-layout.png`
    Optional responsive screenshot showing the app still readable on a smaller viewport.

Recommended style:

- Use dark mode app captures only.
- Prefer clean data, readable metrics, and one strong story per screenshot.
- Avoid exposing real API keys, tokens, account IDs, or personal data.
- Keep file names stable so they can be referenced directly from the main README.
